Project Overview

The aim of the project was to develop a project management system that allows users to create, edit, and manage tasks within a project. The system aimed to provide a comprehensive tool for task management, incorporating features like task assignment, adjacency matrix representation, and domain graph visualisation.

Deliverables

  • Domain Knowledge and Graph: The project included the creation of domain and entity classes in Kotlin to represent tasks as graphs, supporting the Java-based GUI application. This structure allowed users to visualise project dependencies and tasks in a structured manner.
  • Adjacency Matrix Implementation: An adjacency matrix was created in Kotlin, enabling users to view the task dependencies within a project. This matrix was integrated into the GUI, offering a clear and readable representation of the project's structure.
  • Desktop Application with GUI: The team successfully developed a desktop application using Java, which provided users with the ability to create, edit, and delete projects and their tasks through a graphical user interface (GUI).

Technologies Used

  • Java: Used for developing the desktop application with a GUI, which allowed users to interact with the system through buttons for creating, updating, and deleting tasks and projects.
  • Kotlin: Employed to create the domain classes and graphs representing the project tasks. Kotlin was also used to implement the adjacency matrix, which visualises the connections between tasks.
