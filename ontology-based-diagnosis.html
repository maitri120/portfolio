Ontology-Based Expert System For Enhanced Disease Diagnosis

Project Overview

The goal of this project was to develop an advanced expert system that utilises ontology-based knowledge representation to assist in the diagnosis of diseases based on user-reported symptoms. The system was designed to provide accurate and detailed diagnostic suggestions, enhancing the process of disease identification and improving the overall user experience.

Deliverables

I spearheaded the development of a sophisticated expert system integrated with a graphical user interface (GUI) to streamline the diagnosis of diseases from symptoms provided by users. The project combines artificial intelligence with ontological engineering to create a comprehensive and intuitive diagnostic tool.

The system uses medical ontologies to model complex medical knowledge, facilitating detailed symptom-disease mapping. Leveraging the 'Protégé' ontology management tool, I was able to merge and manage intricate ontologies, which significantly enhanced the system’s capability to understand and process a wide range of medical conditions.

The core of the expert system is a rule-based engine that applies backward chaining logic to infer potential diseases based on user inputs. By integrating ontological data, the system achieves improved accuracy and depth in diagnosis.

To ensure ease of use, I developed a user-friendly GUI using Tkinter, which allows users to input symptoms easily and receive diagnostic results. The interface was seamlessly integrated with the expert system, displaying both the diagnosis and actionable health information.

Technologies Used

  • Programming Language: Python
  • Ontology Management:
    • Protege: Used for creating and managing medical ontologies.
    • W3C Web Ontology Management (OWL): Employed for building the knowledge base, representing relationships between diseases and symptoms.
  • Rule-Based System: Implemented using backward chaining logic for disease inference.
  • Graphical User Interface (GUI):
    • Tkinter: Used for creating an intuitive and interactive user interface.
  • ChatBot Framework:
    • Rasa Framework: For potential integration with a chatbot that could assist users in navigating the system.
  • Natural Language Processing (NLP): Utilised for interpreting and managing user inputs.
